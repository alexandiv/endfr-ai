# AGENTS.md

Парсер формата ENDF-6 (ядерные данные). Входные файлы — исходная библиотека
и результат прогона через модуль `reconr` программы `njoy2016`.

## Файлы

- `main.cpp` — весь код парсера (пока в одном файле, `namespace endf`).
- `n-001_H_001.endf` — исходный файл ENDF (H-1).
- `n-001_H_001.reconr` — он же после `njoy2016/reconr` (рабочий вход).
- `CMakeLists.txt`, `CMakePresets.json` — сборка (пресеты `GCC_Debug`,
  `GCC_Release`).
- `.clang-format` — стиль; форматировать обязательно перед коммитом.

## Стиль кода

Используем **минимальное подмножество C++**. Код должен оставаться близким
к C с контейнерами STL. Явно **не использовать**:

- классы с инкапсуляцией (`private`/`protected`), только `struct` с
  публичными полями;
- виртуальные функции, наследование, полиморфизм;
- RTTI (`dynamic_cast`, `typeid`);
- исключения — ошибки через `fprintf(stderr, ...)` + `exit(EXIT_FAILURE)`;
- лямбды, `auto` без нужды, range-based for там, где обычный `for` яснее;
- умные указатели, RAII-обёртки — сырые `FILE*`, обычные массивы;
- операторные перегрузки, конструкторы копирования/перемещения,
  пользовательские деструкторы (пусть генерируются по умолчанию);
- `std::string`, `iostream`, `std::format` — только C stdio и `char[]`.

Разрешено: `std::vector` для динамических массивов, фиксированные типы
`<cstdint>`, структуры с конструктором по умолчанию для инициализации полей.

## Конвенции

- Имена функций/полей/переменных — `snake_case`, константы — `UPPER_SNAKE`.
- Константы формата ENDF (`LINE_WIDTH`, `FIELD_WIDTH` и т.п.) — в namespace.
- Функции объявляются сверху файла, реализации ниже.
- Комментарии допускаются на русском (как в текущем `main.cpp`).
- Форматирование строго по `.clang-format`.
